❓ dúvidas

Perguntas frequentes

Git e GitHub são a mesma coisa? +
Não! Git é uma ferramenta que fica instalada no seu computador. GitHub é um site onde você guarda e compartilha código. É como a diferença entre o Word (programa local) e o Google Docs (na nuvem). Você usa o Git para criar os "saves" e o GitHub para guardar esses saves na internet.
Preciso saber programar pra fazer o curso? +
Não precisa ser dev ninja. Se você já escreveu algum código (qualquer linguagem) e sabe o que é uma pasta no computador, já tá bom. O Git funciona com qualquer tipo de arquivo, inclusive TXT e HTML simples.
O que é um commit, afinal? +
Pensa num commit como um save de videogame com anotação. Você bate git commit -m "adicionei login" e o Git tira uma foto do estado do projeto naquele momento com a sua mensagem. Se der ruim, você volta pra qualquer save anterior.
E se eu bagunçar tudo e quiser voltar atrás? +
Essa é exatamente a graça do Git! Tem vários jeitos: git restore pra descartar mudanças, git revert pra desfazer commits de forma segura, e git stash pro famoso "guarda isso aqui que eu vejo depois". A gente cobre tudo isso no curso.
